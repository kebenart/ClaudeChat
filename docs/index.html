快速开始
几行命令跑起来

把占位符换成你自己的子域名与服务器,详见仓库内《使用指南》。

# 1. 安装并构建npm installnpm run build # → dist/ (Web) + dist-server/ (后端) # 2. 启动(服务只监听环回 127.0.0.1:3001)npm run start # = build + server # 3. 暴露公网:frpc 隧道 → 云端 nginx (TLS)# 浏览器打开 https://cli.example.com:8443/ → 四端连同一地址 # 改完代码,一键重新上线npm run build && launchctl kickstart -k gui/$(id -u)/com.user.claudechat # 4. (可选)让终端里的 claude 也进 IM + 墙内用量代理,一键:node scripts/setup-im-hook.mjs # 注册 hooknode scripts/setup-im-hook.mjs --proxy # 顺带设用量代理

提示:测试期可用 DEV_AUTH_BYPASS 免登录;上线请关闭并启用密码 + TOTP。setup-im-hook 还顺带开启「手机答终端的选择题」与「AI 发图片」;终端 hook / 用量代理 / 选择卡 / 发图片的完整步骤见《使用指南》第 8–10 节。

路线图
Windows 与 Android,在路上

不重写第五、第六个端 —— 直接把已经打磨好的 Web 端打包成桌面 / 移动应用,复用同一套服务枢纽、同步逻辑与账号。

Windows

用 Tauri / Electron 把 Web 端包成 .exe —— 系统托盘常驻、原生通知与红点,连同一个 FRP 地址,无需另写一套界面。

Android

用 Capacitor / TWA 把 Web 端包成 APK —— 系统推送、角标与返回手势,登录沿用四端同一账号,代码几乎零改动。
