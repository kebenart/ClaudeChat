# ClaudeChat 使用指南

> 一个 Claude 专用的 claudecodeui 私有分支:微信式 IM 多端(Web / macOS / iOS / watchOS),
> 家里 Mac 跑服务 + FRP 内网穿透,云服务器做 TLS 终结和公网入口。

> **占位符约定**:文中 `cli.example.com`(你的子域名)、`<SERVER_IP>`(你的云服务器公网 IP)、
> `8443`(公网入口端口)、`<username>` 等请替换成你自己的真实值。**不要把真实域名/IP/密钥提交到仓库。**

---

## 0. 架构总览(先看这张图)

```
                          公网
┌──────────────────────────────────────────────────────────────────┐
│  手机/电脑浏览器、iOS、macOS、Apple Watch                           │
│        │  HTTPS / WSS  →  https://cli.example.com:8443             │
└────────┼───────────────────────────────────────────────────────────┘
         ▼
┌─────────────────────────── 腾讯云 (<SERVER_IP>) ──────────────────┐
│  nginx  :8443  (TLS 终结, WebSocket 升级, 3600s 长超时)              │
│     │  proxy_pass → 127.0.0.1:7080  (仅环回)                         │
│  frps  :7000  (FRP 服务端, 控制通道; 只放行隧道端口 7080 在环回)     │
└─────────────────────────────┬──────────────────────────────────────┘
                              ▲ │  FRP 隧道 (TLS), frpc 主动外拨
                              │ ▼
┌─────────────────────── 家里 Mac (本机) ─────────────────────────────┐
│  frpc        (把本机 127.0.0.1:3001 映射到云端 127.0.0.1:7080)      │
│  app server  node dist-server/server/index.js  →  127.0.0.1:3001    │
│     ├─ Express API  /api/*                                          │
│     ├─ WebSocket    /ws (聊天) /shell (终端) ...                    │
│     └─ 静态托管     dist/ (Web 前端)                                │
│  DB: ~/.cloudcli/auth.db (better-sqlite3)                           │
│  IM 数据来源: ~/.claude/projects/**/*.jsonl → 蒸馏 → im_* 表        │
└─────────────────────────────────────────────────────────────────────┘
```

**要点**
- 服务只监听 **环回 `127.0.0.1:3001`**,不直接暴露公网;公网入口是云端 nginx,经 FRP 隧道回到本机。
- 四端都连同一个地址:**`https://cli.example.com:8443`**(Web 直接浏览器打开,原生端在设置里填这个)。
- 当前**测试阶段**服务端开了 `DEV_AUTH_BYPASS=1`(免登录);上线前必须关掉,改用 密码 + TOTP。

## 1. 本机服务:部署与启动

### 1.1 一次性准备

```bash
cd ~/CODE/claudecodeui-local
npm install                 # postinstall 会自动 patch node-pty,别管它
```

`.env`(`chmod 600`)里**至少**要有 `JWT_SECRET`:

```bash
# .env
JWT_SECRET=<32+ 字节随机十六进制>     # openssl rand -hex 32
```

> ⚠️ **JWT_SECRET 同时用来 AES-GCM 加封每个用户的 TOTP 密钥**。换了它 = 所有人 TOTP 失效,得重新绑定。**别动它。**
> 目前 `.env` 里只设了 `JWT_SECRET`;没设 `HOST`,所以走代码默认 `127.0.0.1`(环回,正是 FRP 方案要的)。

### 1.2 构建

```bash
npm run build          # = build:client (→ dist/) + build:server (→ dist-server/)
npm run build:client   # 只重建 Web 前端 (vite → dist/)
npm run build:server   # 只重建后端 (tsc + tsc-alias → dist-server/)
```

### 1.3 启动方式

| 场景 | 命令 | 说明 |
|---|---|---|
| **生产(手动)** | `npm run server` | 跑 `dist-server/server/index.js`,需先 `npm run build` |
| **生产(一键)** | `npm run start` | = `build` + `server` |
| **开发热更** | `npm run dev` | 后端(tsx)+ vite 前端;开 `http://localhost:5173` |
| **仅后端** | `npm run server:dev` / `npm run server:dev-watch` | tsx 直跑源码(可带 watch) |

### 1.4 开机自启(launchd,推荐——这就是现在的运行方式)

已安装单元:`~/Library/LaunchAgents/com.user.claudecodeui-local.plist`
- 跑 `node dist-server/server/index.js`,`RunAtLoad` + `KeepAlive`(崩了自动拉起)。
- 当前带 `DEV_AUTH_BYPASS=1`(测试免登录)。
- 日志:`/tmp/cclocal.out.log`、`/tmp/cclocal.err.log`。

```bash
# 安装(如果还没装)
install -m 644 deploy/launchd/com.user.claudecodeui-local.plist \
  ~/Library/LaunchAgents/com.user.claudecodeui-local.plist
# 编辑 plist:替换 REPLACE_USER / REPLACE_NODE_PATH(which node)/ REPLACE_REPO_PATH
launchctl load -w ~/Library/LaunchAgents/com.user.claudecodeui-local.plist

# 校验
launchctl list | grep claudecodeui
curl -fsS http://127.0.0.1:3001/health        # 期望 200
```

**改完代码后重新上线**(最常用的两行):

```bash
cd ~/CODE/claudecodeui-local && npm run build
launchctl kickstart -k gui/$(id -u)/com.user.claudecodeui-local
```

## 2. HOST / IP 绑定(`127.0.0.1` vs `0.0.0.0`)

代码默认 `HOST=127.0.0.1`(`server/index.js`)。这决定服务监听哪些网卡:

| `HOST` | 谁能连 | 用途 |
|---|---|---|
| `127.0.0.1`(默认) | **只有本机**(含 frpc、本机浏览器) | ✅ FRP 方案的正确选择——公网入口走 nginx,不直接暴露本机 |
| `0.0.0.0` | 同一**局域网**的任何设备 | 仅当你想**局域网直连**(比如家里平板直接 `http://<Mac的IP>:3001`)才需要 |

```bash
# 临时局域网暴露(仅本次)
HOST=0.0.0.0 npm run server
# 或在 .env 里写 HOST=0.0.0.0(launchd 需在 plist 的 EnvironmentVariables 里加)
```

> 🔒 **安全红线**:`DEV_AUTH_BYPASS=1` 时**绝对不要**配 `0.0.0.0` 或对公网开放——等于裸奔。
> 要么保持 `127.0.0.1`+FRP+TOTP,要么先关 bypass 再考虑 LAN 暴露。

## 3. FRP 内网穿透 + 域名绑定

两台机器:**家里 Mac(跑 frpc)** + **腾讯云 `<SERVER_IP>`(跑 frps + nginx)**。
完整脚本见 `deploy/README.md`,这里是要点。

### 3.1 共享 token(两端必须一致)

```bash
openssl rand -hex 32
```
填进 `deploy/frpc.toml` 和 `deploy/frps.toml` 的 `auth.token`。

### 3.2 Mac 端:frpc

`~/.config/frpc/frpc.toml`(由 `deploy/frpc.toml` 复制并填 token):

```toml
serverAddr = "<SERVER_IP>"      # 腾讯云
serverPort = 7000                 # frps 控制通道
auth.token = "<和 frps 一致>"
transport.tls.enable = true
[[proxies]]
name       = "claudecodeui-local"
type       = "tcp"
localIP    = "127.0.0.1"
localPort  = 3001                 # 本机 app server
remotePort = 7080                 # 云端环回端口(nginx 反代到这里)
```

frpc 已用 launchd 常驻(`com.user.frpc`,现在正在跑)。手动管理:

```bash
brew install frpc
install -m 600 deploy/frpc.toml ~/.config/frpc/frpc.toml      # 然后填 token
frpc -c ~/.config/frpc/frpc.toml                              # 前台验证: "login to server success"
# 自启:
install -m 644 deploy/launchd/com.user.frpc.plist ~/Library/LaunchAgents/com.user.frpc.plist
launchctl load ~/Library/LaunchAgents/com.user.frpc.plist
```

### 3.3 腾讯云端:frps

`/etc/frp/frps.toml`(由 `deploy/frps.toml` 安装):

```toml
bindPort = 7000
allowPorts = [{ start = 7080, end = 7080 }]   # 隧道只在环回暴露 7080
auth.token = "<和 frpc 一致>"
transport.tls.force = true
```

```bash
sudo install -m 600 -o frp -g frp deploy/frps.toml /etc/frp/frps.toml
sudo install -m 644 deploy/systemd/frps.service /etc/systemd/system/frps.service
sudo systemctl enable --now frps
```

> 防火墙/安全组:**只放行入站 `7000/tcp`**;`7080` 必须留在环回(别对外开)。

### 3.4 域名绑定 + nginx(TLS 公网入口)

1. **DNS**:把 `cli.example.com` 的 A 记录指向 `<SERVER_IP>`,等生效。
2. **证书**:`sudo certbot --nginx -d cli.example.com`。
3. **nginx vhost**(由 `deploy/nginx-cli.conf` 安装到 `/etc/nginx/conf.d/cli.conf`):

```nginx
server {
    listen 8443 ssl http2;                 # ← 本部署用 8443(示例文件写的是 443,按需改)
    server_name cli.example.com;
    ssl_certificate     /etc/letsencrypt/live/cli.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/cli.example.com/privkey.pem;
    client_max_body_size 25m;

    location / {
        proxy_pass http://127.0.0.1:7080;  # ← FRP 隧道出口
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;       # WebSocket
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600s;          # Claude 流式长输出
        proxy_send_timeout 3600s;
        proxy_buffering off;
    }
}
```

```bash
sudo nginx -t && sudo systemctl reload nginx
# 安全组同时放行入站 8443/tcp
```

> ⚠️ **端口说明**:四端默认连的是 **`:8443`**(见 watch 默认地址),所以 nginx 要 `listen 8443`,安全组放行 8443。`deploy/nginx-cli.conf` 示例里写的是 `443`,套用时改成 `8443`(或你自己选的端口,并同步改各端地址)。

### 3.5 验证

```bash
curl -I https://cli.example.com:8443/health     # 期望 HTTP/2 200
# frpc 没跑时返回 502;frps 没跑时手机侧 502。
```

## 5. 认证与安全

- **测试阶段**:launchd plist 里 `DEV_AUTH_BYPASS=1` = 关闭一切认证(密码 + TOTP)。**上线前删掉这个 key**,然后 `launchctl kickstart -k ...` 重启。
- **生产**:密码 + TOTP(首登强制绑定)。登录流程:`POST /api/auth/login` → 若 `requiresTotp:true`,再 `POST /api/auth/login/totp` 带 6 位码。
- **JWT_SECRET**:放 `.env`,**不要轮换**(它加封了所有 TOTP 密钥)。
- **TOTP 找回**(丢了验证器 + 恢复码):
  ```bash
  cd ~/CODE/claudecodeui-local && npm run reset-totp -- <username>
  # 之后只用密码登录,会重新引导绑定 TOTP
  ```
- **端口暴露面**:云端只对公网开 `7000`(FRP 控制)和 `8443`(nginx);`7080` 仅环回。本机 `3001` 仅环回。

## 8. IM 终端 hook(让终端里的 claude 也进 IM)

IM 消息是 **hook 驱动**,不是文件监听。从 IM app 发的消息由服务器进程内记录;
你在**终端**里直接跑 `claude`(或别的外部客户端)的会话,只有注册了仓库自带的
hook 才会进 IM hub。hook 在 `UserPromptSubmit`(你的提问)和 `Stop`(回复完成)
两个事件触发,把每个回合 POST 给服务器仅环回的 `/api/im-hook` 端点。

**一键安装(推荐)** —— 自动生成/复用 token、写 plist、合并注册 settings.json、重载 launchd(幂等,先备份 settings.json):

```bash
node scripts/setup-im-hook.mjs            # 仅 hook
node scripts/setup-im-hook.mjs --proxy    # 顺带设 CLAUDE_USAGE_PROXY=http://127.0.0.1:7890(见 §9)
```

它同时注册三个 hook:`UserPromptSubmit`+`Stop`(把终端回合录进 IM),以及 `PreToolUse`(matcher `AskUserQuestion|ExitPlanMode`)的**选择卡 hook**——终端会话遇到选择题/计划确认时,会在所有 IM 端弹出红包式卡片,**你在手机上答**,答案再回传给终端的 claude(阻塞等待,超时 10 分钟)。脚本会保留你已有的其它 hook(如 `rtk hook claude`)。App 会话的选择题仍由服务器进程内处理,不重复。

> 选择卡每个问题都自带一个 **「其他(自己填)」** 选项,选中后弹出输入框,可填自定义答案(单选/多选均支持)。提交后卡片**立即**翻成「已选择 …」(乐观更新),发送失败会显示红色感叹号、点一下可重试。

下面是它替你做的事(需要手动时照做):

1. 生成一个共享 token(任意随机 hex):

   ```bash
   node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"
   ```

2. 写进**服务端** env。在 launchd plist
   (`~/Library/LaunchAgents/com.user.claudecodeui-local.plist`)的
   `EnvironmentVariables` 里加:

   ```xml
   <key>IM_HOOK_TOKEN</key>
   <string>粘贴你的TOKEN</string>
   ```

   plist 改了 env 必须**完整重载**(不能只 `kickstart -k`):

   ```bash
   launchctl bootout gui/$(id -u)/com.user.claudecodeui-local
   launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.user.claudecodeui-local.plist
   ```

3. 在 `~/.claude/settings.json` 注册 hook(保留已有的 hooks;用**同一个** token
   作环境前缀,端点就是 loopback + token 双重校验):

   ```jsonc
   "hooks": {
     "UserPromptSubmit": [
       { "hooks": [{ "type": "command",
         "command": "IM_HOOK_TOKEN=粘贴你的TOKEN node /绝对路径/claudecodeui-local/scripts/im-claude-hook.mjs" }] }
     ],
     "Stop": [
       { "hooks": [{ "type": "command",
         "command": "IM_HOOK_TOKEN=粘贴你的TOKEN node /绝对路径/claudecodeui-local/scripts/im-claude-hook.mjs" }] }
     ]
   }
   ```

hook 永不阻塞 claude(2s 超时、永远 exit 0)。token 只存在 plist 和
`settings.json`,**别提交进仓库**。验证:开个**新**终端跑 `claude`、发一句话,
应当出现在 IM app 里。

## 9. 用量额度(5 小时 / 7 天)与代理

个人中心「用量」卡读你真实的 Claude 账号额度
(`GET https://api.anthropic.com/api/oauth/usage`,OAuth token 来自
`~/.claude/.credentials.json` 或 macOS Keychain)。服务端缓存 **30 分钟**;
手动刷新(iOS 下拉 / Web·macOS 刷新按钮)带 `?force=1`,但对真实上游调用做
**5 分钟下限**节流。

国内该端点被地理封锁(直连 → HTTP 403)。**只**把这一个请求走代理——在 plist 的
`EnvironmentVariables` 里加(按 §8.2 重载):

```xml
<key>CLAUDE_USAGE_PROXY</key>
<string>http://127.0.0.1:7890</string>
```

指向你本地代理(如 Clash `127.0.0.1:7890`)。它**只**作用于用量请求——Claude
SDK 子进程不绑代理,所以代理关了聊天照常。不在墙内就别加这个 key。

## 10. AI 发图片(截图发我)

让 Claude 把图片(如测试结果截图)作为图片气泡推进 IM。你对 Claude 说"截图发我 / 把测试结果截图发我",它产出图片后运行:

```bash
node /绝对路径/claudecodeui-local/scripts/im-send-image.mjs <图片路径> [说明]
```

服务端**校验**(扩展名+magic bytes+≤12MB)并把图拷进受管目录(`~/.cloudcli/im-media/<随机id>.<ext>`,绝不直接对外暴露原路径),记一条 `kind:'image'` 消息广播给各端;客户端经鉴权的 `GET /api/im/media/:id` 取字节渲染。会话由 `CLAUDE_CODE_SESSION_ID`(Claude 自动注入 Bash 环境)确定,token 从环境或 plist 读。
